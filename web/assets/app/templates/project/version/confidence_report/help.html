General Information

Issues are imported from Jira with all their sub-tasks and linked bugs. After importing some information is aggregated and saved.

Columns

The Assignees field shows the aggregated information about assignees.
In simple mode only the developers are shown. In detailed mode (see "Expand Assignees" below) the testing-developeris also shown.

The Status field shows the aggregated issue status.
It may be one of: Open, In progress, Doc, In TBD, In QA, Resolved, Done.

The CL field shows the Confidence Level of the issue.

See "Filtering issues" section below to learn how to filter by assignees, status or CL.

The Impl, Doc, TBD, TC, QA and CR fields are calculated dynamically.
The dash "—" means that appropriate task is not needed.

Updating and saving

To update the issues state click the orange "Update" button and wait for a moment.
All issues will be re-imported. All aggregated fields will be re-calculated and saved.
You can also update each issue separately. Click the update button near the issue key to do so.

The CLR is being saved automatically after any update.
But you have to save additionally after any manual input, i.e. changing the progress, cl, or note.

Importing issues

You can add any story, task or bug from this iteration to the Confidence Levels report.
To do so click the appropriate link after the "Import:".
Already imported issues will be hidden by default. Click the "Show imported" checkbox to show them.

Filtering issues

Type a search keyword to filter issues by key, summary, note, status, progress.
Issues will be filtered on-the-fly as you will enter more than 2 symbols.

Type s: qa or s: tbd or s: %whatever% to filter issues by aggregated status.

Type a: %name% to filter issues by aggregated assignees.

Type t: %name% to filter issues by developer who is responsible for TBD.

Type c: %component% to filter issues by component.

Type tp: %type% to filter issues by type (i.e. story, task or bug).

Type cl: %expr% to filter issues by CL. Examples:
cl: 9 — shows issues with CL = 9.
cl: >6 — shows issues with CL > 6. Also works with "<".
cl: 4-6 — shows issues with CL between 4 and 6 (incl).

You can also filter by multiple statuses, assignees or tester. Just type them separated by comma.
Ex.: s: open, progress, doc or s: tbd, qa.

Manage view

Check Expand Assignees to show detailed information about assignees for the story.

Check Show Subtasks to show sub-tasks in detailed issue view.

Check Show Linked Issues to show linked issues in detailed issue view.
